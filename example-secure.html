Secure Page

This page demonstrates recommended header-based protections and explains why they matter.

Recommended response headers (set on the server)

Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
Content-Security-Policy: default-src 'self'; script-src 'self'; object-src 'none'; frame-ancestors 'none';
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: no-referrer
Permissions-Policy: geolocation=(), microphone=()

Important: These should be delivered as response headers by the server in production — meta tags alone are not sufficient.

Why these help

  • CSP reduces XSS risk by restricting where scripts and other resources can load from.
  • HSTS forces browsers to use HTTPS and prevents downgrade attacks.
  • X-Frame-Options / frame-ancestors prevent clickjacking.
  • X-Content-Type-Options prevents MIME-type sniffing.
  • Referrer-Policy limits leaking URL data to other sites.
